Available DatablockTypes

The different DatablockTypes optionally control special handlings for the respective datablocks.

    Type Description
    DRIVER1 Flash drivers have to be downloaded to an ECU if they do not permanently reside in the ECU (usually they don’t).
    DRIVER1 is the “erase driver”. If Flash DRIVER2 is not used, it also contains the “write driver”.
    DRIVER2 DRIVER2 is the “write driver”.
    DATA The data containing for example all the calibration data for the application (characteristic curves, DTC, thresholds, etc.)

Downloading several drivers is supported. Optionally the drivers may even be different.

Tester Serial Number

The Tester Serial Number entered by the user must be 4 Bytes or 8 Bytes long.
If no Tester Serial Number is entered, then '00000000' will be used.

Download

Logical Blocks

Logical Blocks are addressed by an address.

  • The logical block areas are erased by addresses.
  • The logical block areas are programmed by addresses.

Encryption/Compression

PreCompression and Encryption is supported.

  • PreCompression: Optionally the template can download already compressed flashware. The download of the datablock is marked as compressed download.
  • Encryption: Optionally the template can download already encrypted flashware. The download of the datablock is marked as encrypted download.

Checksum

  • CRC: A CRC file can be selected for every datablock with a 32 bit checksum.
  • Signature: A signature file can be selected for every datablock. The usage of the signature is optional and depends on the security class.

Security Handling

  • A SeedKey.dll is part of the template (only a demo dll).
  • 4 Byte Seed Extension can be used.
  • If no Seed Extension is used, then an ECU specific SeedKey DLL is required.

Communication

Only CAN interface is supported.

CAN

  • CAN ID Type usage: For functional and physical addressing different CAN ID Types (11 bit and 29 bit) may be used.
